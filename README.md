## Commit Philosophy

Repository started Day 1 before any code was written.
Commits follow the convention:

- `init:` — scaffolding and planning
- `task1:` — product scoping
- `task2:` — pipeline development
- `fix:` — corrections and iterations
- `docs:` — documentation

Every commit message explains *what* changed and *why*.
